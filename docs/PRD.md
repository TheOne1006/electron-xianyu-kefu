# 闲鱼客服自动化 PRD

## 1. 核心目的

### 1.1 项目背景

闲鱼二手交易平台客服需求痛点：

- 人工客服成本高、响应慢，难以覆盖海量咨询
- 客户期望快速响应，夜间/节假日无人值守
- 重复性问题（价格咨询、商品状态）占用大量人工精力

本项目通过 AI 助手自动处理客户咨询，实现 24 小时智能客服。

### 1.2 核心价值

| 价值点                | 说明                                      |
| --------------------- | ----------------------------------------- |
| 24 小时自动回复       | 摆脱人工时间限制，夜间节假日照常服务      |
| 意图分类 + 个性化回复 | 根据对话上下文生成专业、友好的回复        |
| 对话历史持久化        | 积累真实对话数据，为后续模型训练提供语料  |
| 人工接管机制          | AI 无法处理时自动标记，确保不出现答非所问 |

### 1.3 范围界定（V1）

**做：**

- 自动检测闲鱼网页未读消息
- AI 自动生成并发送回复
- 对话历史持久化到本地文件（可用于训练数据积累）
- 人工接管自动检测（AI 回复仅含配置字符时触发）
- 商品目录管理 + 文档库管理
- 数据导入导出（全部配置和数据的备份/恢复）
- 日志系统（多进程日志收集、按日期存储、UI 查看）
- 统一 store 数据目录（`app-data/`）
- Webhook 通知测试
- 支付卡片检测与自动发货（固定文本回复 / Webhook 通知）
- 浏览器窗口生命周期管理（状态感知、主窗口联动关闭）
- 快速开始 & Q&A 引导页面

**不做：**

- 商品上下架管理
- 订单系统集成
- 多账号支持
- 统计报表

## 2. 用户故事

| #    | 用户故事                                       | 验收标准                                                             |
| ---- | ---------------------------------------------- | -------------------------------------------------------------------- |
| US-1 | 作为客服，我希望自动回复客户消息               | 收到客户消息后自动生成并发送回复（事件驱动，毫秒级响应）             |
| US-2 | 作为运营，我希望配置 LLM 参数                  | 可在 UI 配置模型、API 地址、API 密钥、人工接管触发字符               |
| US-3 | 作为客服，我希望查看对话历史                   | 可在聊天页面查看历史消息                                             |
| US-4 | 作为客服，我希望手动接管对话                   | 点击停止按钮后立即停止自动回复，不再生成新回复                       |
| US-5 | 作为客服，我希望 AI 无法回复时自动标记人工接管 | 当 AI 回复仅为配置的特殊字符时，系统跳过发送但保持记录，等待人工处理 |
| US-6 | 作为运营，我希望管理商品目录                   | 可在商品页面 CRUD 商品信息，支持自动采集闲鱼商品数据                 |
| US-7 | 作为运营，我希望管理文档库                     | 可在文档页面 CRUD 文档内容，供 Agent 回复时引用                      |
| US-8 | 作为运营，我希望导出/导入全部数据               | 可一键导出所有配置和数据为 JSON 文件，也可从 JSON 文件导入恢复       |
| US-9 | 作为运营，我希望查看系统日志                   | 可在日志页面按日期查看历史日志，实时接收新日志                       |
| US-10 | 作为运营，我希望测试 Webhook 通知             | 在配置页面可点击测试按钮验证 Webhook URL 是否可用                   |
| US-11 | 作为运营，我希望支付后自动发货                 | 检测到支付卡片后，根据商品配置自动发送固定文本或触发 Webhook 通知   |
| US-12 | 作为客服，我希望感知浏览器窗口状态             | 可一键启动/关闭闲鱼浏览器，主窗口关闭时自动联动关闭                 |
