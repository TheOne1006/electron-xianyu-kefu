# Feature Specification: 会话列表 UI 布局优化

**Feature Branch**: `002-conversations-ui-polish`
**Created**: 2026-04-22
**Status**: Draft
**Input**: User description: "ConversationsPage 会话列表 UI 优化：对齐消息数、添加商品主图、截断消息预览、并排展示聊天与详情、可拖拽调整布局"

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 会话列表与聊天内容并排展示 (Priority: P1)

用户打开聊天记录页面后，左侧始终显示会话列表，右侧显示选中会话的聊天消息。点击左侧不同会话时，右侧消息区域即时切换，无需离开消息视图再返回列表。

**Why this priority**: 这是整个功能的核心价值 — 消除当前"列表 / 详情"来回切换的体验断裂，让用户可以快速浏览多个会话。

**Independent Test**: 点击左侧任意会话，右侧立即显示对应聊天内容；点击另一个会话，右侧切换；左侧列表始终可见。

**Acceptance Scenarios**:

1. **Given** 用户在聊天记录页面且未选中任何会话，**When** 用户点击左侧列表中的某条会话，**Then** 右侧区域显示该会话的全部聊天消息，左侧列表保持可见
2. **Given** 用户已选中一个会话正在查看消息，**When** 用户点击左侧列表中的另一条会话，**Then** 右侧消息区域立即切换为新会话的内容，无需返回操作
3. **Given** 用户已选中一个会话，**When** 会话列表自动刷新（10 秒轮询），**Then** 左侧列表数据更新但右侧消息视图不受干扰

---

### User Story 2 - 左右面板可拖拽调整比例 (Priority: P2)

用户可以通过拖拽中间分隔条来调整会话列表与聊天内容之间的宽度比例，以适应不同的查看需求（例如想看更宽的消息气泡或更长的会话名称）。

**Why this priority**: 并排布局后比例固定会限制灵活性，拖拽调整是提升体验的重要补充。

**Independent Test**: 将鼠标移至分隔条位置，按住拖拽可改变左右面板宽度，释放后宽度保持不变。

**Acceptance Scenarios**:

1. **Given** 左右面板并排展示，**When** 用户拖拽中间分隔条向左移动，**Then** 左侧会话列表宽度减小，右侧聊天区域宽度增大
2. **Given** 左右面板并排展示，**When** 用户拖拽中间分隔条向右移动，**Then** 左侧会话列表宽度增大，右侧聊天区域宽度减小
3. **Given** 用户拖拽分隔条，**When** 列表宽度达到最小值（200px），**Then** 分隔条无法继续向左拖动
4. **Given** 用户拖拽分隔条，**When** 列表宽度接近窗口总宽度，**Then** 消息区域保持可见但可能缩至很窄

---

### User Story 3 - 会话列表样式优化 (Priority: P3)

每个会话卡片中的信息排列更加规范：消息条数紧跟在用户名后面、显示商品主图缩略图、最近一条消息预览长度限制在两行以内。

**Why this priority**: 视觉对齐和紧凑度优化是锦上添花，核心功能不受影响。

**Independent Test**: 查看会话列表，确认每个会话项的用户名后紧跟消息条数、右侧有商品缩略图、消息预览不超过两行。

**Acceptance Scenarios**:

1. **Given** 会话列表中有多条会话，**When** 用户查看列表，**Then** 每个会话项的消息条数（"xx 条"）紧接在用户名文字后面，与用户名在同一行对齐
2. **Given** 会话关联了商品（有 itemId），**When** 用户查看列表项，**Then** 会话卡片右侧显示该商品的主图缩略图（圆形或圆角方形）
3. **Given** 会话关联了商品但商品不存在于商品库中，**When** 用户查看列表项，**Then** 显示默认占位图标，不报错
4. **Given** 会话最后一条消息内容很长，**When** 用户查看列表项，**Then** 消息预览最多显示两行，超出部分用省略号截断

---

### Edge Cases

- 商品图片 URL 加载失败时（防盗链、链接失效），应显示默认占位图标而不是破碎图片
- 会话列表为空时，右侧应显示空状态提示，拖拽分隔条仍然可用
- 会话消息只有图片或卡片类型（无文本内容）时，消息预览应显示"[图片]"或"[卡片]"等占位文字
- 窗口尺寸变化时，拖拽分隔条的位置应保持合理的比例关系，不超出边界

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 页面布局必须改为左右并排结构 — 左侧始终为会话列表，右侧为选中会话的聊天消息内容
- **FR-002**: 必须移除当前"选中后隐藏列表、返回按钮切回"的切换机制，改为左侧列表始终可见
- **FR-003**: 左侧面板和右侧面板之间必须有可拖拽的分隔条，用户可自由调整左右宽度比例
- **FR-004**: 分隔条拖拽必须设置最小宽度限制 — 列表侧最小 200px，消息侧自动填充剩余宽度
- **FR-005**: 会话卡片中"xx 条"消息数必须紧接在用户名文字后面显示，不再右对齐
- **FR-006**: 会话卡片右侧必须显示关联商品的主图缩略图
- **FR-007**: 当商品主图无法加载时，必须显示默认占位图标
- **FR-008**: 最近一条消息预览必须限制为最多两行，超出部分用省略号截断
- **FR-009**: 非文本类型消息（图片、卡片）的预览应显示对应的占位文字（如"[图片]"、"[卡片]"）
- **FR-010**: 选中会话后右侧显示消息内容区域，未选中时右侧显示空状态提示
- **FR-011**: 删除会话功能在新的并排布局中必须正常工作

### Key Entities

- **会话卡片 (Session Card)**: 会话列表中的每一项，包含用户名、消息条数、商品主图缩略图、最近消息预览。通过 `itemId` 关联到商品库以获取主图
- **商品 (Product)**: 通过 `itemId` 与会话关联，提供 `mainImageUrl` 用于在会话卡片中展示缩略图
- **可拖拽分隔条 (Resizable Divider)**: 位于会话列表和聊天内容之间的交互控件，支持鼠标按住拖拽改变两侧面板宽度

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 用户从查看一个会话切换到另一个会话的操作步骤从 3 步（返回 → 选择 → 查看）减少到 1 步（直接点击）
- **SC-002**: 所有会话卡片中的用户名与消息条数在同一行、相邻显示，视觉上统一对齐
- **SC-003**: 商品主图缩略图在 1 秒内加载显示（本地缓存图片更快），加载失败时无可见错误
- **SC-004**: 任意长度的消息预览均不超过两行高度，列表整体视觉整齐统一
- **SC-005**: 用户可在 2 秒内完成一次面板宽度调整操作

## Clarifications

### Session 2026-04-22

- Q: 消息区是否有最小宽度限制（原 300px）？ → A: 消息区无固定最小宽度，自动填充列表侧之外的全部剩余宽度

## Assumptions

- 商品主图 URL 通过已有的 `product-store` 查询获得，页面需要从主进程获取商品列表数据用于匹配
- `ChatInfo.itemId` 可以用来匹配 `Product.id` 或商品的某个标识字段来获取 `mainImageUrl`
- 拖拽分隔条使用纯 CSS/JS 实现，不引入第三方拖拽库
- 默认分隔条初始位置：左侧列表 280px（与当前设计一致）
- 删除会话、发送消息等已有功能在新布局中保持原有行为不变
